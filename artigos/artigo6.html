A internet e a Web

A internet está presente no nosso dia a dia há vários anos, mas você sabe como ela funciona? Bem… se você deseja hackear algum sistema, entender como um dispositivo se comunica com outro é fundamental, sendo a internet um dos principais meios de comunicação do mundo atual é indispensável entender o seu funcionamento.

Sendo assim, a seguir vamos entender a essência da internet:

O que é a internet?

A internet é uma rede que conecta os dispositivos (computador, smartphone, tablet, etc) do mundo, permitindo assim que seja compartilhado milhões de terabytes de dados, como por exemplo e-mail, fotos, vídeos, etc. Normalmente tratamos a internet e a Web como sinônimos, mas são conceitos diferentes, pois a Web (World Wide Web, o famoso WWW) é apenas um dos serviços que funciona sobre a infraestrutura da internet.

É importante notar que existem diversos serviços que funcionam com base na infraestrutura da internet, como por exemplo o e-mail e IRC. Fazendo uma analogia simples, a internet é igual a uma estrada gigante que conecta o mundo, e a Web seria como uma das portas que permite o acesso a essa estrada. Mas como as informações são transportadas pela internet? Como essas informações não são enviadas para o lugar errado?

Antes de responder essas perguntas precisamos entender o conceito de cliente e servidor.

Clientes e Servidores

Os clientes são os dispositivos que consomem o conteúdo que está armazenado nos servidores, sendo que os servidores são os computadores responsáveis por armazenar os dados, como por exemplo sites, aplicativos, banco de dados. Ou seja, quando você acessa um site, você é o cliente, e o servidor é o que envia o conteúdo do site a você.

Vamos imaginar que você quer acessar o site www.exemplo.com.br.

O seu navegador irá fazer uma solicitação ao servidor DNS pedindo o endereço IP da URL www.exemplo.com.br, em seguida o navegador (em posse do endereço IP), irá fazer uma solicitação (request) ao servidor do site, pedindo uma cópia da página web, e o servidor irá responder (response) a solicitação enviando o conteúdo para que o mesmo seja baixado no seu dispositivo para que assim você possa acessá-lo.

A comunicação entre o cliente e o servidor do site será feita usando o protocolo HTTP. Essa comunicação é transportada usando o modelo TCP/IP.

Para ficar mais fácil de entender, vamos fazer uma analogia:

Vamos imaginar que você tem uma lista de doces para comprar, mas você não conhece a cidade, então você faz o seguinte:

Você pesquisa o endereço da loja de doces (o servidor DNS pesquisa o endereço IP do site)

Ao encontrar o endereço da loja, você pega um carro e vai até a loja com a sua lista de compras (no caso o carro é igual ao modelo TCP/IP, ou seja, transporta algo)

Ao chegar na loja, você precisa falar o mesmo idioma do vendedor, para a comunicação acontecer (o protocolo HTTP e o idioma do cliente e do servidor)

Então você volta para a casa com os doces (o conteúdo do site é enviado ao cliente).

DNS

O DNS (Servidor de Nome de Domínio - Domain Name Servers) é o responsável por traduzir a URL para um endereço IP. Sendo o IP um endereço numérico único, ou seja, cada dispositivo conectado à internet possui um endereço IP diferente.

Como por exemplo, a URL:

https://www.google.com.br/

Será pesquisada no servidor DNS é será retornado o endereço IP: 216.58.222.14

HTTP

Como mencionado anteriormente o HTTP (Protocolo de Transferência de Hipertexto - Hypertext Transfer Protocol) é o protocolo que o cliente usa para conversar com o servidor.

Métodos HTTP

Os métodos HTTP definem o que o servidor deve fazer com um determinado recurso.

Principais métodos HTTP: GET

O método GET é usado para solicitar algum recurso ao servidor.

HEAD

O método HEAD é usado para solicitar algum recurso ao servidor (igual ao método GET) mas sem o corpo da resposta.

POST

O método POST é usado para enviar arquivos, dados e formulários HTML ao servidor.

DELETE

O método DELETE é usado para remover um determinado recurso.

Códigos de resposta HTTP

Os códigos de resposta HTTP revelam se uma determinada solicitação foi concluída com sucesso.

Os códigos HTTP são divididos em cinco classes:

100-199 : Respostas de informação.

200-299 : Respostas de sucesso

300-399 : Redirecionamentos

400-499 : Erros do cliente

500-599 : Erros do servidor

Cabeçalhos HTTP

Os cabeçalhos HTTP são usados para transmitir informações adicionais ao cliente ou servidor.

Principais cabeçalhos HTTP Host

O cabeçalho Host é usado para especificar o nome de domínio do servidor. No exemplo da requisição anterior o cabeçalho Host possui o valor www.exemplo.com.br.

Connection

O cabeçalho Connection é usado para especificar se uma conexão irá ficar aberta ou não depois da primeira solicitação ao servidor.

O Connection possui dois valores:

keep-alive : esse valor define que a conexão é persistente, ou seja, permite que sejam feitas requisições futuras ao mesmo servidor.

close : esse valor especifica que o cliente ou o servidor deseja fechar a conexão.

Accept

O cabeçalho Accept é usado para especificar quais tipos de conteúdo o cliente entende.

Alguns dos valores do Accept são:

Accept: text/html : Aceita apenas texto html.

Accept: image/* : Aceita qualquer tipo de imagem, como por exemplo image/png, image/gif.

Accept: */* : Aceita tudo.

Accept: text/html, application/xhtml+xml, application/xml;q=0.9, */*;q=0.8 : Esse valor é o padrão para requisições de navegação.

O valor do Accept será retornado na resposta do servidor, no cabeçalho Content-Type.

User-Agent

O cabeçalho User-Agent é usado para especificar qual sistema operacional, fornecedor ou versão o cliente está usando.

O valor desse cabeçalho identifica o navegador do cliente, junto com outras informações.

Accept-Encoding

O cabeçalho Accept-Encoding é usado para especificar qual codificação o cliente aceita.

Alguns valores desse cabeçalho, são:

ccept-Encoding: gzip : Esse valor define o formato de compreensão como Lempel-Ziv coding (LZ77), com CRC de 32-bits.

Accept-Encoding: * : Esse valor define que é aceitável qualquer codificação de conteúdo. Esse é o valor padrão.

O valor Accept-Encoding será retornado na resposta do servidor, no cabeçalho Content-Encoding.

Server

Esse cabeçalho revela qual é o software usado pelo servidor. No nosso exemplo o servidor e o Apache.

TCP/IP

Anteriormente, foi apresentado de forma breve o modelo TCP/IP. Nesta seção será explicado de forma mais profunda esse modelo.

O TCP/IP (Protocolo de Controle de Transmissão e Protocolo de Internet - Transmission Control Protocol e Internet Protocol) é um conjunto de protocolos que define como a comunicação entre dois ou mais computadores acontece.

O modelo TCP/IP é dividido em quatro camadas: Aplicação, Transporte, Rede e Interface de rede.

Aplicação

A camada de aplicação é a primeira na arquitetura TCP/IP, sendo responsável por realizar a comunicação entre os programas e os protocolos de transporte. Os protocolos DNS e HTTP funcionam na camada de aplicação, outros protocolos que pertencem a essa camada, são:

FTP (File Transfer Protocol) : O FTP é usado para enviar arquivos entre dois computadores.

SMTP (Send Mail Transfer Protocol) : O SMTP é usado para enviar e receber e-mails.

Transporte

A camada de Transporte é responsável por definir um padrão de qualidade de serviço para que a conexão seja estabelecida, mantida e encerrada. Os principais protocolos usados dentro dessa camada são:

UDP (User Datagram Protocol) : O protocolo UDP é usado para transferir dados, que não necessite de garantia de integridade ou confiabilidade. O UDP é adequado para se usar em situações em que a velocidade de transmissão dos dados é mais importante que a qualidade. Como por exemplo em videoconferências e jogos.

TCP (Transmission Control Protocol) : O protocolo TCP é orientado a conexão, ou seja, garante a integridade e ordem dos dados enviados. O TCP usa um three way handshake para estabelecer uma conexão confiável.

Para você entender o three way handshake imagine que o host A quer estabelecer uma conexão com o host B, para que isso aconteça, será necessário que:

O host A envie um pacote de sincronização SYN (SYNchronize) ao host B.

O host B irá responder com um pacote SYN-ACK (SYNchronize-ACKnowledgment).

É por fim o host A irá enviar um pacote ACK (ACKnowledge) para o host B, estabelecendo assim a conexão.

Rede

A camada de Rede, também conhecida como camada de Internet, é responsável por adicionar o cabeçalho no pacote, especificando o endereço IP de origem e destino

A camada de Rede ou Internet é a responsável por adicionar o cabeçalho no pacote de dado recebido da camada de Transporte, onde além de outros dados de controle, será adicionado o endereço IP fonte e o endereço IP de destino.
