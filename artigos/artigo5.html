Introdução ao Linux - Conceitos básicos

Olá pessoal, tudo tranquilo?

Neste artigo você irá aprender os conceitos básicos do Linux, que inclui os comandos mais utilizados para manipulação de arquivos, permissões de acesso, compactação e descompactação de arquivos, entre várias outros assuntos.

O que é Linux?

O Linux é um sistema operacional, assim como o Windows e o MacOS, com a diferença de ter o seu código aberto, o que permitiu o desenvolvimento de várias distribuições Linux (ou simplesmente distro). Em termos mais técnicos o Linux é só o kernel, o kernel é o que gerencia recursos do computador. O kali linux foi feito usando esse kernel, por isso nós falamos que o kali é uma distribuição linux.

Existem várias distribuições Linux, como o Ubuntu e o Linux Mint. O que diferencia o kali delas é o fato dele ser voltado para a área de hacking, por isso várias ferramentas já vem instaladas nele, mas se você quiser usar outra distribuição e instalar as mesmas ferramentas é totalmente possível.

O que é uma distribuição Linux?

Uma distribuição Linux ou distro é simplesmente um sistema operacional que usa alguma parte do Kernel Linux.

Exemplos de distribuições Linux:

  • Linux Mint
  • Ubuntu
  • Debian
  • Kali Linux

Interpretador de comandos

Antes de ver os comandos do terminal, é importante conhecer alguns conceitos básicos.

O interpretador de comandos (ou shell) é o responsável por executar os comandos fornecidos pelo usuário, ou seja, o shell é o responsável por fazer a intermediação entre o sistema operacional e o usuário. O shell padrão na maioria dos sistemas Linux é o Bash (Bourne Again Shell). Existem outros shells como: Sash; Sh; Zsh; entre outros.

O shell pode ser dividido em dois grupos:

  • CLI (Command Line Interface) : são shells que possuem a interface baseada em linha de comando.
  • GUI (Graphical User Interface) : são shells que usam interface gráfica, como por exemplo o Windows Explorer.

Manipulação de diretório e arquivos:

Uma coisa muito importante que você deve lembrar é que tudo no Linux são arquivos. O GNU/Linux é Case Sensitive, ou seja, ele diferencia letras maiúsculas de minúsculas.

ls

O comando ls é usado para listar o conteúdo de um diretório, por exemplo nomes de arquivos e diretórios. Diretório são pastas.

Com a opção -l é listado mais detalhes, como data de criação, permissões.

Com a opção -a é listado os arquivos ocultos. Arquivos ocultos são arquivos que começam com um ponto.

Existem várias outras opções, para conhecê-las digite: ls --help

pwd

O comando pwd imprime o caminho (localização) do diretório, começando da raiz.

cd

O comando cd é usado para mudar de diretório. Esse comando é essencial para navegar na estrutura de diretórios

O Linux tem uma estrutura de diretório definida pelo FHS (Filesystem Hierarchy Standard). Essa estrutura organiza os arquivos de forma hierárquica. Essa estrutura é importante para ajudar a manter a compatibilidade entre as várias distribuições Linux.

Para acessar o diretório raiz digite: cd /

O diretório home é onde os usuários ficam: cd home

No caso, sou o usuário kali, que é o usuário padrão.

Quando entramos no diretório kali, vemos os arquivos do usuário:
cd kali
ls

Podemos acessar o diretório kali usando um caminho absoluto, que é simplesmente digitar todo o caminho a ser percorrido.

Então, para acessar o diretório kali, vai ficar assim:

cd /home/kali

Podemos usar o pwd para confirmar a nossa localização:

pwd

Vamos voltar novamente para o diretório raiz:

cd /

Bem, é possível, é até mais eficiente usar só o comando cd para voltar ao diretório home:

cd

mkdir

O comando mkdir cria um diretório (pasta). Um diretório é igual uma pasta pode ser usado para armazenar arquivos ou outros diretórios.

É possível criar mais de um diretório com um único comando:

mkdir exemplo1 exemplo2

rmdir

Esse comando remove (apaga) um diretório. O diretório deve estar vazio para ser apagado.

rmdir exemplo

touch

Esse comando é usado para criar arquivos vazios, ele também pode ser usado para mudar a data e hora que um arquivo foi criado.

A opção -t é usada para modificar a data e a hora. Essa opção usa o formato Mês, Dias, Horas e opcionalmente o ano e segundos.

Para criar um arquivo vazio digite:

cat

Esse comando mostra o conteúdo de um arquivo.

cat exemplo/texto.txt

A opção -n mostra o número das linhas enquanto o conteúdo do arquivo é mostrado:

cat -n exemplo/texto.txt

tee

O comando tee envia a saída de um comando para o terminal e para um arquivo.

ls -l | tee exemplo.txt

Agora se listarmos novamente os arquivos: ls

Veremos que o arquivo exemplo.txt foi criado. E se olharmos dentro do arquivo exemplo.txt veremos o resultado do ls -l

Em resumo, o pipe ( | ) envia a saída do comando ls -l para o tee, que vai exibir o resultado no terminal e vai gravar o resultado no arquivo exemplo.txt

cat exemplo.txt

É possível também enviar a saída de um comando só para um arquivo usando o sinal maior que. Por exemplo:

ls > listagem.txt

Vai enviar o resultado do comando ls para o arquivo listagem.txt (que será criado), vamos usar o ls

Bem, o arquivo foi criado, agora vamos verificar o conteúdo do arquivo:

cat listagem.txt

tac

O comando tac mostra o conteúdo do arquivo na ordem inversa.

rm

Esse comando é usado para apagar arquivos, também pode ser usado para apagar diretórios e sub-diretórios vazios ou não.

Vamos apagar o arquivo teste.txt:

A opção -v mostra os arquivos que estão sendo apagados.

Dentro do diretório teste:

exemplo1.txt exemplo2.txt exemplo3.txt

A opção -r é usada para remover arquivos em sub-diretórios, esta opção também pode ser usada para remover sub-diretórios.

Dentro do diretório teste:

exemplo teste01.txt teste02.txt

O comando cp é usado para copiar arquivos.

Copia o arquivo teste.txt para dentro do diretório /exemplo

mv

O comando mv é usado para mover ou renomear arquivos e diretórios.

Renomea o arquivo teste.txt para teste1.txt

Move o arquivo teste.txt para o diretório /exemplo

Coringas:

Os coringas são usados para definir um ou mais arquivos ou diretórios de uma só vez. Existem 4 tipos de coringas, são eles:

* - asterisco

? - ponto de interrogação

[ ] - colchetes

{ } - chaves

Os coringas são usados em conjunto com os comandos, por exemplo:

Vamos criar 5 arquivos: teste1.txt, teste2.txt, teste3.txt, teste4.txt e teste5.txt, você pode fazer isso de pelo menos duas formas:

Criando um arquivo de cada vez, usando o comando touch, por exemplo:

touch teste1.txt touch teste2.txt

E assim por diante.

Também é possível criar os arquivos de outra forma bem mais eficiente, usando algum coringa, por exemplo:

touch teste{1,2,3,4,5}.txt

Agora, usamos o ls para ver os arquivos é… ai estão

Agora vamos listar somente os arquivos teste1.txt, teste2.txt e teste3.txt, para isso vamos usar o coringa [ ]

ls teste[1-3].txt

Legal, agora vamos criar mais dois arquivos, só que agora com a extensão .py

touch exemplo{6,7}.py

Vamos listar os arquivos que começam com exemplo e terminam com .py.

ls exemplo?.py

O ponto de interrogação (?) substitui um caractere no caso são os números 6 e 7.

Bem, agora vamos conhecer o último coringa.

Vamos listar os arquivos primeiro: ls

Agora, vamos apagar todos os arquivos que terminam em .txt:

rm *.txt

O asterisco (*) faz referência a um nome completo de um arquivo ou diretório, pode ser lido como tudo.

No caso do exemplo, seria apague tudo que termine em .txt

Comandos diversos:

find

Quando o find é usado sozinho ele irá listar todos os arquivos, diretórios e subdiretórios inclusive os ocultos.

Agora, vamos procurar somente pelos arquivos do diretório exemplo

find / -name "*.txt"

A barra ( / ) significa que a busca pelos arquivos vai começar a partir do diretório raiz ( / ), o -name define o que deve ser procurado, no caso são os arquivos com a extensão .txt

Bem, agora vamos pesquisar novamente por arquivos com a extensão .txt

find . -name "*.txt"

Dessa vez a busca pelos arquivos vai começar a partir do diretório atual, é por isso que usamos o ponto (.)

Bem, agora vamos pesquisar pela palavra password, começando do diretório raiz

find / -name "password"



grep

O comando grep é usado para procurar por um padrão de caracteres específico.

Por exemplo, o seguinte comando vai pesquisar pela palavra backup no arquivo passwd que fica dentro do diretório /etc

grep backup /etc/passwd

Agora, vamos criar um arquivo chamado usuarios.txt

nano usuarios.txt

Dentro do arquivo vamos colocar os nomes:

Ricardo, João, Fernanda, Pedro, ricardo, joão, fernanda, pedro

Para salvar use as teclas: Ctrl + x depois pressione a tecla y e de enter

Vamos usar o cat para ver o conteúdo do arquivo:

Bem, quando pesquisamos pelo nome Ricardo

grep “Ricardo” usuarios.txt

Veja que só foi listado o nome Ricardo com o R maiúsculo, isso acontece porque o comando grep diferencia letras maiusculas de minusculas, para resolver isso se usa o opção -i

grep -i "Ricardo" usuarios.txt

Também é possível usar o grep com outros comandos usando o pipe ( | )

cat usuarios.txt | grep -i "Ricardo"

O pipe pega a saída do comando cat, que no caso é os nomes de usuários e passa como entrada para o comando grep, que faz a busca pelo nome Ricardo.

nl

O comando nl mostra o número de linhas e o conteúdo do arquivo.

Vamos criar o arquivo texto

Dentro dele vamos digitar algumas letras: A B C D E F G H

nl -fn texto.txt

more

O comando more faz a paginação de arquivos e da entrada padrão.

O comando more é usado para visualizar arquivos de texto, exibindo uma tela por vez caso o arquivo seja grande. Ele é semelhante ao comando cat, mas exibe o conteúdo em páginas, tornando mais fácil para ler arquivos grandes. Por exemplo, quando exibimos o conteúdo de um arquivo com o cat ele mostra tudo de uma vez:

cat /etc/passwd

Mas quando usamos o more o conteúdo é dividido em partes:

more /etc/passwd

Para continuar vendo o conteúdo do arquivo pressione a tecla de espaço ou enter. Para sair pressione a tecla q.

Também, é possível usar o pipe ( | )

cat /etc/passwd | more

less

Esse comando faz a paginação de arquivos igual ao more, com a diferença de que o less permite voltar à página anterior, usando a tecla seta para cima e a seta para baixo para avançar.

less /etc/passwd

tail

O comando tail mostra as linhas finais de um arquivo. Por padrão, as 10 últimas linhas são mostradas caso não seja especificado o número de linhas a ser exibidas com a opção -n.

tail usuarios.txt tail -n 11 usuarios.txt

time

O comando time mede o tempo que um processo (programa) leva para ser executado.

Esse é o tempo gasto que o find leva para procurar pela palavra passwd.

echo

O comando echo é usado para mostrar mensagens na tela.

echo "Hacking"

wc

O comando wc mostra o número de palavras, bytes e linhas de um arquivo.

wc usuarios.txt

Para mostrar a quantidade de linhas se usa a opção -l

wc -l usuarios.txt

seq

O comando seq imprime uma sequência de números, os números podem ser inteiros ou reais (com pontos decimais).

seq 10

Vai imprimir do 1 ao 10 no terminal.

É possível especificar dois números, onde o primeiro vai ser o número que vai dar início a contagem é o segundo vai definir o número final:

seq 5 10

Vai imprimir na tela o número 5 até o 10.

Como você pode ver a sequência de números tem sido incrementada de 1 em 1, mas é possível definir o valor do incremento.

Por exemplo:

seq 3 0.5 10

Vai imprimir do número 3 até o 10 com o incremento de 0.5, ou seja vai de 3 até 10 com um aumentando de 0.5

Também é possível imprimir uma sequência para trás, basta usar um incremento negativo:

seq 5 -1 2

É possível definir um formato para ser impresso usando a opção -f

Por exemplo:

seq -f '##%g##' 1 5

Se tentarmos imprimir números com ponto flutuante, assim:

seq -f '## %g ##' 1.0 5.0

Não vai dar certo, porque é necessário usar o %f no lugar do %g

seq -f '##%f##' 1.0 5.0

Também é possível usar um caractere qualquer para separar os números usando a opção -s

seq -s ':' 4

Esse comando compara dois arquivos de qualquer tipo (texto ou binário).

Dentro dele vamos colocar:

Um exemplo

diff

O comando diff também é usado para comparar dois arquivos.

Bem, a diferença entre o cmp e o diff é que o diff só compara arquivos texto.

diff -B -i teste1.txt teste2.txt

O diff vai fazer a comparação entre os arquivos teste1.txt e teste2.txt levando em consideração que a opção -B define que linhas em em branco devem ser ignoradas e a opção -i especifica que se deve ignorar diferenças entre letras maiúsculas e minúsculas.

Se usarmos o comando diff para fazer a comparação, não será retornada uma mensagem de erro, mas a mensagem não irá mostrar onde está a diferença entre os arquivos.

Compactação e descompactação de arquivos:

Fazer a compactação de arquivos é útil quando é necessário transferir vários arquivos, enviando assim só o arquivo compactado.

tar

O comando tar é usado para fazer a compactação e descompactação de arquivos. Os arquivos compactados com o tar normalmente ficam com a extensão .tar

Vamos compactar os arquivos usuarios.txt senhas.txt no arquivo dados.txt

tar -cf dados.tar usuarios.txt senhas.txt

A opção -c é usada para criar um novo arquivo .tar

Já a opção -f define o nome do arquivo, que no caso é dados.tar. Vamos ver se foi compactado os arquivos: ls

Para descompactar o arquivo basta usar o -x no lugar do -c

tar -xf dados.tar

Manipulação de conta:

O Linux é um sistema multiusuário, ele permite que vários usuários usem simultaneamente o sistema, sem que haja conflitos entre eles.

O usuário mais poderoso do sistema é o root, ele não possui limitações e é responsável por gerenciar todo o sistema. O comando sudo dá acesso aos poderes do root, um uso comum do sudo é quando se precisa instalar um programa ou simplesmente atualizar todo o sistema.

Por exemplo, se tentarmos atualizar o sistema com o comando:

apt-get update

Recebemos um erro, pois é necessário ter permissão do usuário root, para resolver o erro basta usar o comando sudo na frente:

sudo apt-get update

Será necessário digitar a senha do usuário root, que no caso é kali já que eu estou usando o usuário padrão.

Aprenderemos ainda nesse vídeo mais sobre o comando apt-get.

A senha do usuário (padrão) root é: kali



Permissões de acesso:

As permissões de acesso servem para definir quem pode ler, escrever ou executar um arquivo ou diretório.

Para vermos quais permissões um arquivo tem, basta usar o: ls -l

O primeiro caractere que no caso é o d significa que é um diretório, os arquivos usam um traço ( - )

Os três próximos caracteres, que no caso é da segunda letra a quarta, indicam as permissões de acesso do dono do arquivo. O dono é o usuário que criou o arquivo.

Da quinta a sétima letra mostra as permissões de acesso do grupo. Um grupo são vários usuários diferentes que têm acesso ao mesmo arquivo.

Da oitava a décima letra mostra quais são as permissões de acesso para os outros usuários. Outros usuários são todos os que não são donos nem fazem parte de um grupo que tem acesso ao arquivo.

Bem, mas o que são essas letras?

A letra r (Read) simboliza a permissão de leitura para um arquivo, se for um diretório seria a permissão para listar o seu conteúdo usando o ls por exemplo.

A letra w (Write) representa a permissão de escrita ou gravação, ou seja, permite criar ou alterar o conteúdo de um arquivo.

A letra x (Execute) simboliza a permissão de execução de um arquivo, por exemplo executar um script, se for um diretório permite o uso do comando cd.

O traço ( - ) significa que não se tem permissão.

Um arquivo pode ter três tipos de usuários:

Dono: é o usuário que criou o arquivo.

Grupo: Um grupo permite que vários usuários diferentes tenham acesso ao mesmo arquivo.

Outros: Todos os usuários que não são donos nem fazem parte de um grupo que tem acesso ao arquivo.

Entender as permissões de arquivos é muito importante para fazer a escalação de privilégios.

chmod

O comando usado para definir as permissões é o chmod. Ele pode alterar as permissões do grupo e dos outros usuários. Para mudar o dono do arquivo se usa o comando chown.

Para usar o chmod é necessário especificar três perguntas:

Primeiro: para quem está sendo atribuído às permissõe; Segunda: que mudanças estão sendo feitas, está adicionando ou removendo permissões; E terceira: qual permissão está sendo modificada

Parece complexo, né?, mas não se preocupe, com os exemplos tudo ficará mais claro.

Para usar o chmod é necessário especificar três perguntas:

Primeiro: para quem está sendo atribuído às permissões; No caso, está sendo atribuído para o arquivo usuarios.txt. Segunda: que mudanças estão sendo feitas, está adicionando ou removendo permissões; No caso, estamos removendo. E terceira: qual permissão está sendo modificada. No caso, estamos removendo a permissão de leitura.

Existem duas formas de se alterar as permissões, usando o modo literal (simbólico) ou o modo octal.

Primeiro vamos ver o modo literal.

Para responder a primeira pergunta do chmod usando o modo literal, usamos os valores:

u : representa o dono do arquivo.

g : representa o grupo.

o : representa os outros usuários.

a : representa todos os itens citados anteriormente.

Para responder a segunda pergunta, usamos os valores:

- : o sinal de subtração é usado para remover permissão.

+ : o sinal de adição é usado para adicionar uma permissão sem remover a existente.

= : o sinal de igual é usado para adicionar uma permissão e remover a existente.

Para responder a terceira pergunta usamos os valores que vimos anteriormente:

r : representa a permissão de leitura.

w : representa a permissão de escrita.

x : representa a permissão de execução.

É possível ver as permissões usando a interface gráfica.

Agora vamos ver alguns exemplos:

Primeiro, vamos criar o arquivo usuarios.txt

Dentro dele, vamos colocar

Ricardo : senha 1234

João : senha 9043

Salve o arquivo.

Vamos ver as permissões e o conteúdo do arquivo: ls -l

cat usuarios.txt

Agora, vamos impedir que não seja possível ler o conteúdo do arquivo:

chmod u-r,g-r,o-r usuarios.txt

Agora, se vermos novamente as permissões do arquivo, veremos que foram removidas: ls -l

Se tentarmos vermos o conteúdo do arquivo:

cat usuarios.txt

Receberemos uma mensagem dizendo que a permissão foi negada.

Agora, vamos dar a permissão de leitura novamente:

chmod ugo+r usuarios.txt

Modo octal

É possível alterar as permissões usando o modo octal, que é basicamente usar números no lugar das letras.

O número 4 representa a permissão de leitura (r).

O número 2 representa a permissão de escrita (w).

O número 1 representa a permissão de execução (x).

É possível fazer a soma dos números para atribuir mais de uma permissão por vez. Por exemplo, para um arquivo ter permissão de leitura, escrita e execução, você usa a soma de 4 + 2 + 1 que dá 7.

Vamos fazer o mesmo exemplo de antes, só que dessa vez usando números:

chmod -444 usuarios.txt

Agora, se tentarmos ver o conteúdo do arquivo, obteremos o mesmo erro de permissão negada de antes.



adduser

Esse comando é usado para criar um novo usuário ou grupo.

Vamos criar o usuário Ricardo, com a senha 2194

sudo adduser ricardo

Por padrão quando o usuário é criado o nome do grupo fica igual ao do usuário, é possível mudar isso, mas não entraremos em detalhes agora.

O comando sudo é usado para executar um comando como superusuário, ou seja, como root. Por padrão, quando se usa o sudo é necessário digitar a senha do usuário que se está usando, no caso, estamos usando o usuário padrão, que é o kali, que tem como senha kali.

Por exemplo, quando tentamos ver o conteúdo do arquivo shadow, recebemos uma mensagem informando que não temos permissão para acessar o arquivo.

O arquivo /etc/shadow é um arquivo de texto que contém informações sobre as senhas dos usuários do sistema.

cat /etc/shadow

Agora, se usarmos o comando sudo na frente:

sudo cat /etc/shadow

Conseguimos ver o que tem dentro do arquivo.

Perceba que quando usamos o comando sudo, após o comando ser executado perdemos o privilégio do root.

Para mudar isso, podemos acessar o usuário root de forma permanente usando:

sudo su

Agora para executarmos qualquer comando que precise de privilégios do root não é necessário usar mais o sudo na frente.

Por exemplo:

cat /etc/shadow

Não é recomendado usar o usuário root a todo tempo, por questões de segurança, pois você pode acabar executando algum comando que prejudique o sistema operacional.

Para sair do usuário root basta usar o comando: exit



Baixando e instalando programas:

Agora, vamos aprender como baixar e instalar programas usando a linha de comando.

apt

O apt é uma ferramenta de linha de comando usada para gerenciar pacotes das distribuições Linux.

Mas, o que eu quero dizer com gerenciar pacotes?

Quando eu falo gerenciar pacotes me refiro a atualizar, instalar, configurar e remover programas. Os programas podem ser ferramentas como o Nmap, SQLmap, ou uma linguagem de programação como o Python.

As principais opções do apt são:

Para atualizar a lista de pacotes disponíveis use:

apt update

O comando apt update atualiza a lista de pacotes para pegar as atualizações disponíveis nos repositórios que estão listados em /etc/apt/sources.list.

Para instalar as atualizações que estão na lista use o comando:

apt upgrade

O comando apt upgrade instala a versão mais recente de todos os pacotes.

A lista com as versões fica em /etc/apt/sources.list, nessa lista fica as fontes de onde serão baixados os programas, e também será usado para buscar por atualizações.

É recomendado que se use o apt update antes de se usar o apt upgrade, pois assim você atualiza a lista com as versões mais recentes para serem instaladas.

Para instalar um pacote ou programa use: apt install pacote

Vamos fazer um exemplo, iremos instalar a linguagem Python: apt install python

Agora, podemos ver se a instalação foi bem sucedida: python --version

Para remover um programa do sistema se usa: apt remove pacote

Por exemplo para remover o python use: apt remove python

O git clone é usado para clonar um repositório do GitHub. O git clone não é um comando do Linux, ele é um comando do Git, que é um sistema de controle de versão distribuído, usado principalmente para desenvolvimento de software.

Bem, o Git possui vários outros comandos que vou mostrar futuramente. Por enquanto, vou dar um exemplo do uso do git clone:

git clone https://github.com/aboul3la/Sublist3r.git

Arquitetura do Linux

A arquitetura do linux pode ser subdividida em: Hardware; Kernel; Biblioteca de funções padrão; Shell; Aplicações

O hardware é o componente que representa a camada física, por exemplo o disco rígido, CPU, etc.

O kernel é o responsável pela interface entre o hardware e as aplicações, o kernel é responsável por gerenciar os recursos disponíveis, por exemplo gerenciar a memória.

A biblioteca de função padrão é a responsável por permitir o acesso a recursos através da execução de chamadas feitas por processos. Algumas funções padrão são a open, read, write e close.

Assim como eu falei no início do artigo, a Shell é a responsável por executar os comandos fornecidos pelo usuário.

A camada de aplicações é a que está mais próxima do usuário, alguns exemplos são, navegadores, gerenciador de arquivos, etc.
